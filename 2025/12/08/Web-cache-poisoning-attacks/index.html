WEB缓存欺骗

发表于2025-12-08|更新于2025-12-12|WEB安全
|浏览量:

不积硅步，无以至千里。

为了提高网页显示速度并减轻服务器负载，许多公司都依赖缓存机制。
该系统能够临时存储经常请求的资源（例如图像、脚本或 HTML 页面），以便在后续请求中更快地提供这些资源。

除了有助于提升用户体验外，缓存还在搜索引擎优化 (SEO) 中发挥着关键作用，因为优化的加载时间可以对搜索引擎排名产生积极影响。
然而，为了在不损害安全性的前提下充分利用缓存的优势，必须了解其工作原理和不同类型的缓存。这种了解尤为重要，因为缓存管理中的漏洞若利用不当，可能会为诸如 Web 缓存投毒等可怕的攻击铺平道路。

什么是缓存以及它是如何工作的

缓存系统在优化网站性能方面发挥着关键作用。它允许临时存储某些资源，以减少对服务器的冗余调用，并加快页面加载速度。

缓存主要分为两种类型：私有缓存和共享缓存。

  • 私有缓存是特定于某个用户的。它通常存储在浏览器中，包含特定于该用户会话或个人资料的元素。这种类型的缓存绝对不能共享，因为它可能包含敏感信息。
  • 共享缓存用于存储多个用户共用的内容，例如静态 HTML 文件、JavaScript 脚本、CSS 样式表或图像。该缓存可以在基础设施的不同层级进行管理，特别是通过代理、反向代理或内容分发网络 (CDN)。其目的是减轻源服务器的负载，并加速大规模资源的交付。

正是这种共享缓存成为了网络缓存投毒攻击的主要目标。如果攻击者成功将恶意响应注入缓存，就可以将其提供给众多用户，从而扩大攻击的影响范围。

网络生态系统中还有其他类型的缓存：OPcache（用于执行 PHP 脚本）和数据库缓存。由于它们不受 Web 缓存中毒的直接影响，因此本文不作讨论。

了解 HTTP 标头在缓存管理中的作用

HTTP 标头在缓存管理中扮演着至关重要的角色。它们使参与请求处理的各个各方（浏览器、服务器、代理、CDN 等）能够知道如何存储、验证或拒绝响应。其中，对缓存而言，最重要的就是标Cache-Control头。

Cache-Control 和 Age 标头

根据RFC 9111，Cache-Control用于指定适用于请求或响应路径上所有缓存的指令。

该标头可以出现在 HTTP 响应中，以指示如何缓存响应；也可以出现在 HTTP 请求中，以指示客户端希望中间缓存如何处理响应。

另一个有用的标头是Agecache_time，它指示响应已被缓存的时间。该值以秒为单位，可用于判断资源是否仍然“新鲜”或即将过期。
